1.Basic Git flow
mkdir git-practice
cd git-practice
git init

touch README.md
git status
git add README.md
git commit -m "Initial commit with README"

echo "This is my Git practice repository" >> README.md
git add README.md
git commit -m "Update README file"

git log
 
Explanation 
mkdir & cd → Create and enter project folder

git init → Initialize empty Git repository

touch README.md → Create README file

git status → Check file state

git add → Stage file

git commit → Save changes with message

echo >> → Append text to README

git log → View commit history

2.Branching And Merging
git branch feature
git checkout feature

touch feature.txt
echo "Feature branch work" > feature.txt
git add feature.txt
git commit -m "Add feature file"

git checkout main
git merge feature
git branch -d feature

Explanation
git branch feature → Create new branch

git checkout feature → Switch to feature branch

Work is done independently

git merge feature → Merge feature branch into main

git branch -d feature → Delete branch after merge

3.Conflict Resolution
git checkout -b branch1
echo "Line from branch1" > conflict.txt
git add conflict.txt
git commit -m "Branch1 change"

git checkout main
git checkout -b branch2
echo "Line from branch2" > conflict.txt
git add conflict.txt
git commit -m "Branch2 change"

git checkout main
git merge branch1
git merge branch2

Explanation
Conflict happens because same file modified in different branches

Manually resolve and commit

4.Github Collaboration
git clone https://github.com/your-username/sample-repo.git
cd sample-repo

git checkout -b my-feature-branch
echo "My contribution" >> README.md
git add README.md
git commit -m "My contribution"

git push origin my-feature-branch
Fork repository on GitHub

Clone fork locally

Create feature branch

Make contribution

Push branch to GitHub

Create Pull Request on GitHub

5.Advanced Github Commands
git init
touch file1.txt file2.txt
git add .
git commit -m "Add multiple files"

touch .gitignore
echo "node_modules/" >> .gitignore
git add .gitignore
git commit -m "Add gitignore file"

git stash
git stash pop

git tag -a v1.0 -m "First release"
git push origin main
git push origin v1.0

Explanation
git add . → Add all files

.gitignore → Ignore unwanted files

git stash → Temporarily save uncommitted changes

git stash pop → Restore stashed changes

git tag -a → Create version tag
